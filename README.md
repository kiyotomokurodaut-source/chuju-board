# 中学受験 併願計画ボード 2027(定期スクレイピング付き)

首都圏(1都3県+茨城)の私立・国立・公立中高一貫 **272校** の入試日程・説明会・優遇制度・塾別合格実績を
集約し、志望校を選ぶと日程競合を自動表示するサイトです。GitHub Actionsが定期的にWeb検索ベースで各校の
最新情報を出典URL付きで取得し、`data/` に保存 → サイトに自動反映します。

## 収録校とデータの埋まり方(全272校)

- **主要50校**: 入試日程(2026年度実績ベースの想定)と偏差値めやす入りで即使えます
- **公立中高一貫22校**: 都内・神奈川は2/3適性検査を初期値として収録
- **その他約200校**: 名称・種別・所在地・最寄り駅のみ確定情報として収録し、
  日程・偏差値は捏造せず「未取得」と表示。**定期スクレイピングまたはAIボタンが公式情報を
  出典付きで取り込むと、カード・カレンダー・併願自動作成の対象に自動で組み込まれます**
  (偏差値は四谷80等をAIが取得し、サピックス相当に約−7換算して判定に使用)
- 学校の追加・削除は `scraper/schools.json` に1行足すだけ(idは任意の一意な文字列)

### 272校の初回取り込みと運用コスト

1. Actionsの手動実行で `full=1` を指定 → 全校を一括取得(1校あたり約$0.05〜0.15、**初回計$15〜40**)
2. 平常時は既定(週2回×25校、全校一巡に5〜6週間)
3. 説明会・出願シーズン(9〜1月)は cron を毎日+`BATCH_SIZE: "40"` に上げると全校一巡が約1週間
   (目安 月$80前後。予算に応じて調整)

## 仕組み

```
GitHub Actions(火・金 6:00 JST)
  └─ scraper/fetch.mjs
       ├─ Anthropic API + Web検索で各校の情報を構造化取得(出典URL必須)
       ├─ 前回データと比較して差分を data/changelog.json に記録
       └─ data/summary.json ほかを更新してコミット
            └─ Netlify(またはGitHub Pages)が自動再デプロイ
                 └─ index.html が data/*.json を読み込んで表示
```

- 取得失敗時は前回データを保持します(壊れた情報で上書きしない)。
- 1回の実行で `BATCH_SIZE`(既定25)校を巡回し、次回は続きから。既定設定では**週1回、全50校が一巡**します。
- 差分(説明会の追加など)は「更新履歴」タブに表示され、予約開始の見逃し防止に使えます。

## 併願プラン自動作成(v2で追加)

「併願プラン自動作成」タブで、性別・持ち偏差値(サピックス基準 / 四谷大塚合不合は約−7で自動換算)・
エリア・学校タイプ・大学付属の希望を入れると、**攻め/バランス/堅実の3パターン**を1月前受け〜2月後半まで
日程競合なしで自動生成します。

- 「マイ併願プラン」に入れた学校は**固定校**として必ず組み込み、残り枠を埋めます
- 自宅の最寄り駅を入れると、各校へのGoogleマップ経路リンクを表示(学校カードにも反映)
- 各パターンは**HTMLファイルとして保存**でき、LINE・メールで家族に共有できます
- 慶應SFC・中等部などの二次日程は仮押さえして重複可能性を警告します

競合判定は「割当可能性」ベースです。複数回入試のある学校は別の回に逃がせるため、
どの組み合わせでも全校を受けきれない場合にだけ「競合」と表示します。

## リアルタイム上書きの仕組み

1. 定期スクレイピング(または手動AIボタン)が公式情報を出典付きで取得
2. 確認できた入試日程は**カレンダー・カードの想定値を自動で上書き**し、「✓確認」(出典リンク)と「公式情報反映済み」バッジを表示。全回そろえば完全置換、一部のみなら未確認の回を⚠想定のまま残す部分マージ
3. ページは**5分ごと+タブ復帰時**に新データを確認し、更新があればその場で再描画してトースト通知(リロード不要)

更新の速さの上限はスクレイピング頻度です。説明会シーズン(9〜11月)は
`.github/workflows/scrape.yml` の cron を毎日(例: `0 21 * * *`)に上げるのがおすすめです。

## X(Twitter)速報タブ

「#2027年組」などの定番リアルタイム検索、プラン中の学校ごとの最新投稿検索(Xの「最新」タブを直接開く)、
任意アカウント/リストのタイムライン埋め込み(Xの仕様変更で表示されない場合は検索リンクにフォールバック)を提供します。
X APIは有料のため使わず、検索リンク+公式埋め込みウィジェットのみで構成しています。

## セットアップ(10分)

1. このフォルダをGitHubリポジトリとしてpush
2. リポジトリの **Settings → Secrets and variables → Actions** で
   `ANTHROPIC_API_KEY` を登録(https://console.anthropic.com で発行)
3. **Actions** タブでワークフローを有効化 → 「定期スクレイピング」を **Run workflow** で手動実行
   (`full=1` を指定すると初回に全校を一括取得)
4. Netlifyでこのリポジトリを連携(publish directory はルート `.` / netlify.toml が自動適用)
   - 手動更新ボタンもデプロイ先で使う場合は、Netlifyの環境変数にも `ANTHROPIC_API_KEY` を設定
   - GitHub Pagesでも動作します(その場合、手動更新ボタン用のNetlify Functionのみ無効)

## 運用の調整

| 変更したいこと | 場所 |
|---|---|
| 実行頻度 | `.github/workflows/scrape.yml` の `cron`(説明会シーズンの9〜11月は毎日に上げるのがおすすめ) |
| 1回あたりの学校数 | 同ファイルの `BATCH_SIZE` |
| 学校の追加・削除 | `scraper/schools.json`(サイト側も同ファイルを読むため、ここを直すだけで両方に反映) |
| 手動更新の悪用対策 | Netlify環境変数 `REFRESH_TOKEN` を設定するとトークン必須になる(`netlify/functions/refresh.js`) |

## コストの目安

Web検索($10/1,000回)+ Sonnetのトークン代で、**1校あたり約$0.05〜0.15**。
既定(週1回全校一巡)で **月$15〜25程度**。頻度を上げる場合はこの比例で増えます。

## データの信頼性について

- 取得情報にはすべて出典URLが付き、出典が確認できない情報は保存しない設計です。
- それでもAI取得には誤りが混ざり得ます。**出願・予約の最終確認は必ず各校の一次情報で**行ってください。
- `data/` の変更はすべてgitコミットとして残るため、いつ何が変わったかを後から追跡できます。

## ファイル構成

```
index.html                     サイト本体(単一ファイル・data/を自動読込)
scraper/schools.json           学校マスタ(50校: 日程想定・偏差値めやす・タグ)
scraper/fetch.mjs              定期取得スクリプト(依存パッケージなし・Node 20)
.github/workflows/scrape.yml   定期実行(火・金 6:00 JST)+ 手動実行
netlify/functions/refresh.js   手動更新ボタン用APIプロキシ(キーを隠す)
netlify.toml                   Netlify設定
data/                          自動生成(summary.json / changelog.json / schools/*.json / state.json)
```

## ローカルでの動作確認

```bash
MOCK=1 node scraper/fetch.mjs          # APIキーなしでパイプラインを検証
ANTHROPIC_API_KEY=sk-... FULL=1 node scraper/fetch.mjs   # 実データで全校取得
python3 -m http.server 8000            # http://localhost:8000 でサイト確認
```
